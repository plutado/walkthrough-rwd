What is responsive web design?

Let's first examine each word.

What's the most important word of the three?

The most important word in the phrase

Responsive Web Design

is Web

Why?

Because 'web' is the medium we're 'designing' for, and 'responsive' is just an adjective that describes the kind of web we're building.

I believe that very soon, we’ll no longer say Responsive Web Design, we will simply call it Web Design
The key to creating a successful
responsive web design
is understanding the web.

The web is powerful and unique,
so we should leverage it's advantages.

but also...

We should discard any outmoded mental models we've borrowed from other mediums that fail to serve us well on the web.

Let's talk about the other mediums

The order of appearance:

  • PRINT
  • PHOTOS
  • RADIO
  • TELEVISION
  • THE WEB
When television was in it's infancy,
most of the stars came from radio, and the first programs borrowed their formats from radio, such as radio plays, variety shows, and orchestra concerts.

The radio plays in particular weren't 'movie versions' of popular plays - these were literally radio actors, filmed in front of microphones, reading.

That was TV!
We were now able watch people read!
(instead of just hearing them on the radio)

It took a few years for creators to experiment and leverage the advantages that television offers that are not accessible (or even possible) to do on radio.

On the web, we still face a similar story.

When I was developing newspaper sites in 2007 - 2011, everyone felt confident where "above the fold" and "below the fold" existed.

The idea of "above the fold" was adopted from print. Above the fold means all of the content a user can see on their screen without scrolling down. This came from the idea that real newspapers are folded when you buy them - the top half that users can see without unfolding the paper was literally "above the fold".


Our users had a smaller range of monitor resolutions. As long as their monitors were at least 1024px wide, they got the optimal user experience our industry provided, and we could make reasonable assertions (within a couple hundred pixels), of where "above the fold" was on their screens.

Before the web needed to be

responsive,

"print and graphic design thinking" worked pretty well.



But those days are over.

The fact is:

You shouldn't use photoshop to design a responsive web site.

(because photoshop is for static print design)

The prevalence of smart phones and iPads during the emergence of responsive web design had an unexpected and unfortunate consequence...

With "graphic design thinking" dominating our habits, we merely shifted the erroneous idea of a 'page', and began instead to refer to 'device layouts'.

How does my site look in the:

Mobile Layout?

Tablet Layout?

Wait, do you mean portrait or landscape?

Desktop Layout?

Our designers will just create 3 different PSDs, one for each breakpoint!

If we implement those in pixel perfect fashion, our work here is done!

It's taken a log time for us to realize what happens between
breakpoints is just as important as what happens at the breakpoints.

A responsive site that is designed
for the browser


should also be designed
in the browser.

Forget about pixel perfection at
idealized breakpoints or devices.

Aim instead for
perfect responsiveness.

As the web has become more and more

responsive,

a unique characteristic of the web has become more obvious:

Using graphic design tools to build
responsive sites no longer works


in a medium where device
resolutions are no longer predictable.

When some mobile phones have larger resolutions than some tablets, and some tablets have larger resolutions than desktop monitors, not to mention the hybrids devices: phablets (the phone tablet), and the netbook (not quite a tablet, not quite a desktop), using 'device labels' as responsive guides are basically meaningless.

If you think about it, the idea of
having 'device layout' constructs
define your responsive design is silly.

Using graphic design tools to build
responsive sites no longer works

in a medium where pages
no longer exist.

All we have is the viewport
(the area on the screen that users can currently see).

But we have no control over viewport resolutions of the devices used by
visitors to access our sites.

So what is Responsive Web Design?

Responsive Web Design is a methodology for building websites without favor towards specific devices, platforms, or resolution sizes, as such that the website is optimized at all viewport widths, for all devices, in all platforms and that utilize a modern browser.
Effective responsive Web Design has to be a "content-first" approach, where content components are be developed to be fully responsive at any viewport width on it's own merits, regardless of whether the breakpoints used are or are not in concert with other components or happen at device specific resolutions.

Let's talk about media queries

Transition Styles

You can select from different transitions, like:
Cube - Page - Concave - Zoom - Linear - Fade - None - Default

Themes

Reveal.js comes with a few themes built in:
Default - Sky - Beige - Simple - Serif - Night
Moon - Solarized

* Theme demos are loaded after the presentation which leads to flicker. In production you should load your theme in the <head> using a <link>.

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the background.

Custom Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


            Reveal.addEventListener( 'customevent', function() {
            console.log( '"customevent" has fired' );
            } );

Slide Backgrounds

Set data-background="#007777" on a slide to change the full page background to the given color. All CSS color formats are supported.

Image Backgrounds

<section data-background="image.png">

Repeated Image Backgrounds

<section data-background="image.png" data-background-repeat="repeat" data-background-size="100px">

Background Transitions

Pass reveal.js the backgroundTransition: 'slide' config argument to make backgrounds slide rather than fade.

Background Transition Override

You can override background transitions per slide by using data-background-transition="slide".
